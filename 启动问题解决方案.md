# Spring Boot 启动错误解决方案

## 错误信息
```
Invalid bean definition with name 'fileInfoMapper' defined in file [...]: Invalid value type for attribute 'factoryBeanObjectType': java.lang.String
```

## 解决方案

### 方案1：清理并重新编译（推荐）

1. **在IDEA中执行：**
   - 点击菜单 `Build` → `Clean Project`
   - 点击菜单 `Build` → `Rebuild Project`
   - 删除 `target` 目录（如果存在）
   - 重新运行项目

2. **或者使用Maven命令：**
   ```bash
   cd Spring_Boot/HomeService_Platform
   mvn clean
   mvn compile
   mvn spring-boot:run
   ```

### 方案2：更新Maven依赖

1. 在IDEA中，右键点击 `pom.xml`
2. 选择 `Maven` → `Reload Project`
3. 等待依赖下载完成
4. 重新编译运行

### 方案3：如果上述方案都不行

暂时注释掉 `FileInfoMapper`（因为目前没有被使用）：

1. 在 `HomeServicePlatformApplication.java` 中，将 `@MapperScan` 改为：
   ```java
   @MapperScan(basePackages = "org.example.homeservice_platform.mapper", 
               excludeFilters = @ComponentScan.Filter(type = FilterType.ASSIGNABLE_TYPE, 
                                                      classes = FileInfoMapper.class))
   ```

2. 或者直接删除 `FileInfoMapper.java` 文件（如果暂时不需要文件上传功能）
